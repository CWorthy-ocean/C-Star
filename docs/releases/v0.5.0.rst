v0.5.0
------

.. note:: 
    This release is currently in development


Breaking Changes
~~~~~~~~~~~~~~~~

- Deprecated/removed `CSTAR_ORCH_REQD_ENV` environment variable
- Modified parameters to `DagStatus.__init__`
- Removed `workplan-monitor` CLI command
- Removed `dag_runner.attach_to_run` (duplicates `process_plan` functionality)
- The flag constant is removed (`#428 <https://github.com/CWorthy-ocean/C-Star/pull/428>`_)
- Modified the interpretation of `Status.is_running` to exclude `Unsubmitted` (`#461 <https://github.com/CWorthy-ocean/C-Star/pull/461>`_)
- Rename env var group name constants to omit leading underscore (`#462 <https://github.com/CWorthy-ocean/C-Star/pull/462>`_)
- The `roms` executable is now executed in the `work` subdirectory instead of `output` (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)
- The output location in the runtime settings (`roms.in`) file is now _always_ modified to place outputs in the output directory (since we no longer run ROMS in the output directory) (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)
- `ServiceConfiguration`, `JobConfig` have been relocated to `cstar.entrypoint.config` from `cstar.entrypoint.worker.worker` (`#503 <https://github.com/CWorthy-ocean/C-Star/pull/503>`_)
- `BlueprintRequest` deprecated in favor of `XRunnerRequest` (`#506 <https://github.com/CWorthy-ocean/C-Star/pull/506>`_)
- `BlueprintRequest` has been deprecated and removed. (`#510 <https://github.com/CWorthy-ocean/C-Star/pull/510>`_)
- `--stage` CLI parameter for `cstar blueprint run` is no longer supported (`#510 <https://github.com/CWorthy-ocean/C-Star/pull/510>`_)
- `deep_merge` is a pure method and no longer changes the inputs (`#498 <https://github.com/CWorthy-ocean/C-Star/pull/498>`_)
- The `-d` short `dry-run` argument of `cstar workplan run` is removed to avoid a conflict. (`#515 <https://github.com/CWorthy-ocean/C-Star/pull/515>`_)
- The `output-dir` argument of `cstar workplan run` now uses `--o` and `--output` for consistency (previously `-d` and `--output-dir`). (`#515 <https://github.com/CWorthy-ocean/C-Star/pull/515>`_)
- ROMS-specific blueprint, runner, and transforms have been moved to align with the demo applications (now contained in `cstar.applications.roms_marbl`) (`#519 <https://github.com/CWorthy-ocean/C-Star/pull/519>`_)
- The ROMS-specific `SimulationRunner` has been replaced by the use of the generic `BlueprintRunner` and can no longer be invoked with the same entrypoint (`#519 <https://github.com/CWorthy-ocean/C-Star/pull/519>`_)
- Core contracts for application building (e.g. `Transform`, `XRunner`, `RunnerRequest, etc) have been relocated  `cstar.applications.core` (`#519 <https://github.com/CWorthy-ocean/C-Star/pull/519>`_)
- Deprecated the `Registrar` (`#519 <https://github.com/CWorthy-ocean/C-Star/pull/519>`_)
- Deprecated unused `ComputingPlatform` model (`#519 <https://github.com/CWorthy-ocean/C-Star/pull/519>`_)
- The `WorkplanTransformer` no longer receives transforms to the `__init__` method (`#523 <https://github.com/CWorthy-ocean/C-Star/pull/523>`_)
- Use of `{{path: <step-name>}}` in templates is no longer supported. Use `{{work_dir: <step-name>}}` instead (`#522 <https://github.com/CWorthy-ocean/C-Star/pull/522>`_)
- The `RomsMarblBlueprint` model has been updated. Old blueprints may be upgraded by using `cstar blueprint migrate <blueprint-path>` (`#521 <https://github.com/CWorthy-ocean/C-Star/pull/521>`_)
- `JobFileSystemManager` attributes have been modified for clarity and naming consistency (`work_dir`->`run_dir`, `root`->`root_dir`, `output_dir`->`working_dir`). (`#521 <https://github.com/CWorthy-ocean/C-Star/pull/521>`_)
- Sample blueprints are updated to the latest schema version. (`#521 <https://github.com/CWorthy-ocean/C-Star/pull/521>`_)
- the `name` of a task is now the step name (not `step.safe_name`). using `cstar workplan status` on old outputs will result in a key error (`#532 <https://github.com/CWorthy-ocean/C-Star/pull/532>`_)
- The _monitor loop_ has been removed after scheduling a run. `cstar workplan log ...` should be used in it's place (`#546 <https://github.com/CWorthy-ocean/C-Star/pull/546>`_)
- `StateRepository` class now comprises functions from `cstar.orchestration.state` (`#546 <https://github.com/CWorthy-ocean/C-Star/pull/546>`_)
- Env var constants missing correct type annotations are ignored. (`#554 <https://github.com/CWorthy-ocean/C-Star/pull/554>`_)
- All system contexts have been renamed (`#558 <https://github.com/CWorthy-ocean/C-Star/pull/558>`_)
