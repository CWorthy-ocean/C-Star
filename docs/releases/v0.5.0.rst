v0.5.0
------

.. note:: 
    This release is currently in development


Breaking Changes
~~~~~~~~~~~~~~~~

- Deprecated/removed `CSTAR_ORCH_REQD_ENV` environment variable
- Modified parameters to `DagStatus.__init__`
- Removed `workplan-monitor` CLI command
- Removed `dag_runner.attach_to_run` (duplicates `process_plan` functionality)
- The flag constant is removed (`#428 <https://github.com/CWorthy-ocean/C-Star/pull/428>`_)
- Modified the interpretation of `Status.is_running` to exclude `Unsubmitted` (`#461 <https://github.com/CWorthy-ocean/C-Star/pull/461>`_)
- Rename env var group name constants to omit leading underscore (`#462 <https://github.com/CWorthy-ocean/C-Star/pull/462>`_)
- The `roms` executable is now executed in the `work` subdirectory instead of `output` (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)
- The output location in the runtime settings (`roms.in`) file is now _always_ modified to place outputs in the output directory (since we no longer run ROMS in the output directory) (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)

New features
~~~~~~~~~~~~

- Enable user supplied variables for templating from `cstar workplan run ...`
- The orchestrator writes a sentinel file to disk to dynamically alter dependencies (`#459 <https://github.com/CWorthy-ocean/C-Star/pull/459>`_)
- Add trace log level to reduce log frequency during `Workplan` execution (`#460 <https://github.com/CWorthy-ocean/C-Star/pull/460>`_)
- Persist `WorkplanRun` record for every invocation of the orchestrator for run history and lookup (`#461 <https://github.com/CWorthy-ocean/C-Star/pull/461>`_)
- Add `cstar workplan monitor --run-id <run-id>` for reattaching to a running `Workplan` (`#461 <https://github.com/CWorthy-ocean/C-Star/pull/461>`_)
- Add `nesting_info` block to RomsMarblBlueprint (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)
- Create work-dir symlinks to predetermined paths for CDR and nesting info files (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)
- If no `Makefile` is provided in the `compile_time` additional code block, ROMS will copy and use the appropriate `Makefile` from the ROMS repo (`#465 <https://github.com/CWorthy-ocean/C-Star/pull/465>`_)
- Add automated release-note updater, utilizing PR template categories (`#484 <https://github.com/CWorthy-ocean/C-Star/pull/484>`_)
- Locally-launched steps now generate a `.out` log file consistent with slurm-launched steps (`#493 <https://github.com/CWorthy-ocean/C-Star/pull/493>`_)
